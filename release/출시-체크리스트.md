# 오늘쪽지 뽑기 — 출시 체크리스트 (따라만 하면 됨)

코드는 준비 끝났어요. **콘솔에서 값 3종류만 받아 붙여넣으면** 바로 배포할 수 있어요.
아래를 위에서부터 순서대로 하면 됩니다.

---

## 0. 준비물 (한 번만)

- 앱인토스 개발자센터 가입: https://developers-apps-in-toss.toss.im/
- 터미널에서 CLI 로그인
  ```
  npx @apps-in-toss/cli login
  ```

---

## 1. 앱 만들고 `appName` 받기

1. 개발자센터 → **앱 만들기** → 이름 `오늘쪽지 뽑기` 로 생성
2. 생성되면 **앱 ID**(영문 슬러그, 예: `todaynote-ab12`)가 나와요 → 복사
3. `granite.config.ts` 열어서 아래 줄을 바꿔요
   ```ts
   appName: 'today-note',            // ← 여기를
   appName: '콘솔에서_받은_앱ID',     // ← 이렇게 (예: 'todaynote-ab12')
   ```

## 2. 아이콘 올리고 URL 받기

1. 개발자센터 → 앱 설정 → **아이콘 이미지 업로드** (`release/app-icon-600.png` 사용 가능)
2. 업로드하면 `https://static.toss.im/appsintoss/...` URL 이 나와요 → 복사
3. `granite.config.ts` 의 아이콘 줄을 바꿔요
   ```ts
   icon: 'https://static.toss.im/appsintoss/placeholder-today-note.png',  // ← 이 줄을
   icon: '콘솔에서_받은_아이콘_URL',                                        // ← 이렇게
   ```

## 3. 광고 지면 4개 만들고 `adGroupId` 받기

1. 개발자센터 → **수익화 / 광고** → **보상형(리워드) 광고 그룹**을 4개 생성
   (심층 리포트 / 카드 저장 / 다시 뽑기 / 궁합 — 용도별로 구분해두면 나중에 성과 보기 편해요)
2. 각각 발급된 **adGroupId** 4개를 복사
3. `src/lib/ads.ts` 의 `AD_GROUPS` 를 실제 값으로 바꿔요
   ```ts
   export const AD_GROUPS = {
     detail: 'REPLACE_REWARD_DETAIL',   // ← 심층 리포트용 adGroupId
     saveImage: 'REPLACE_REWARD_SAVE',  // ← 카드 저장용 adGroupId
     retry: 'REPLACE_REWARD_RETRY',     // ← 다시 뽑기용 adGroupId
     compat: 'REPLACE_REWARD_COMPAT',   // ← 궁합용 adGroupId
   };
   ```
   > 광고를 아직 안 붙일 거면, 4개를 전부 같은 하나의 adGroupId 로 채워도 돼요.

---

## 4. 값 제대로 채웠는지 확인

```
npm run check:release
```
- `✅ 출시 준비 완료` 가 뜨면 통과.
- ❌ 가 뜨면 어느 값이 아직 placeholder 인지 알려줘요 → 그 파일만 다시 고치면 됨.

## 5. 빌드하고 검토 올리기

```
npm run verify        # 타입체크 + 테스트 + 빌드 + 출시가드 (한 번에)
ait build             # .ait 번들 생성 (dist 기반)
ait deploy            # 앱인토스 콘솔로 검토 요청 업로드
```
- `ait deploy` = "만든 앱을 토스에 '검토해주세요' 하고 올리는 것". 사람이 검수 후 승인되면 토스 앱 안에서 켜져요.
- 콘솔에서 **QR/샌드박스**로 실기기(내 폰 토스앱)에서 먼저 눌러보고 올리는 걸 권장.

## 배포 전에 실기기에서 꼭 한 번 볼 것 (샌드박스)

코드로는 다 됐지만, 아래는 실제 토스 앱 안에서만 최종 확인돼요:
- [ ] 보상형 광고가 실제로 뜨고, **다 본 뒤에만** 리포트/카드가 열리는지
- [ ] 결과/궁합 **카드 저장**이 토스 갤러리에 실제로 저장되는지
- [ ] 공유(친구에게 보내기)가 토스 공유창으로 뜨는지
- [ ] 상단 안전영역(노치) 여백이 정상인지

문제 없으면 그대로 승인 요청하면 됩니다.
