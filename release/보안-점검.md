# 보안 점검 기록 (2026-07-13)

적용 스킬: [addyosmani/agent-skills — security-and-hardening](https://github.com/addyosmani/agent-skills)
(OWASP Top 10 / 시크릿 / 의존성 감사 체크리스트를 이 앱의 배포 맥락에 맞춰 판정)

## 앱의 보안 특성

서버·로그인·결제·자유 입력이 없는 클라이언트 전용 미니앱이라 공격 표면 자체가 작다.
저장하는 값은 전부 선택형(띠·별자리·기분·기록)이고 개인정보/자유 텍스트를 받지 않는다.

## 통과 항목

| 항목 | 상태 |
|---|---|
| XSS 표면 (`innerHTML`·`dangerouslySetInnerHTML`·`eval`·`document.write`) | 없음 — React 자동 이스케이프만 사용 |
| 시크릿 하드코딩 | 없음 (adGroupId 는 REPLACE_ placeholder, 시크릿 아님) |
| CSP | `script-src 'self'`, `object-src 'none'`, `form-action 'none'` 등 엄격 잠금 (index.html meta) |
| referrer 유출 | `no-referrer` |
| 소스맵 노출 | 빌드에 소스맵 없음 (`sourcemap:false`) |
| 콘솔/디버거 잔류 | vite esbuild `drop: console,debugger` |
| localStorage 입력 검증 | `loadSavedPeople` 등 스키마 검증 후 사용 (조작된 값 무시) |
| 에러 내부정보 노출 | ErrorBoundary 는 친화 문구만 표시, 상세는 분석 이벤트로만 (300자 절단) |
| CI 잠금 설치 | `npm ci` (락파일 고정) |
| 광고 보상 무결성 | `rewarded`/`unsupported` 만 해제, dismissed/failed 는 잠금 유지 (테스트로 고정) |

## 연기 항목 (사유 + 재검토일)

### npm audit 32건 (critical 1 · high 13 · moderate 4 · low 14)

- **전부 `@apps-in-toss/web-framework`(토스 공식 프레임워크)의 내부 빌드 툴체인**에서 나온다
  (@babel/core sourceMappingURL 파일 읽기, fastify 등 — 벤더의 CLI/개발서버 의존성).
- **도달 가능성 판정: 런타임 미도달.** 배포 번들(dist)에 해당 패키지 코드가 포함되지 않음을
  grep 으로 확인했다. 취약점은 빌드 머신에서 신뢰된 벤더 CLI 를 실행할 때만 관련된다.
- **수정 불가:** 최신 버전(2.10.6)까지 동일하게 플래그되고, 토스 배포(`ait build/deploy`)에
  이 프레임워크가 필수라 교체 대안이 없다.
- **조치:** 벤더 업데이트 추적. **재검토일: 2026-08-13** (또는 web-framework 다음 마이너 릴리스 시).
- CI 에 `npm audit` 게이트를 넣지 않은 이유: 수정 불가한 벤더 광고로 항상 실패해
  게이트로서 의미가 없음. 위 재검토일에 수동 재확인한다.

### npm audit signatures

- 이 개발 환경의 프록시에서 레지스트리 서명 다운로드가 차단돼 실행 불가.
- **조치:** 로컬(제약 없는 네트워크)에서 배포 전 1회 실행 권장: `npm audit signatures`
