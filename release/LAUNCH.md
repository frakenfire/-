# 🚀 내일쪽지 뽑기 — 출시 절차서

이 문서 순서대로만 하면 출시됩니다. 코드는 전부 준비돼 있고,
**계정 만들기와 콘솔 제출만 사람 손이 필요**해요. (예상 소요: 반나절 + 검수 대기 최대 3영업일)

---

## 1단계. 계정 준비 (10분)

1. https://developers-apps-in-toss.toss.im 접속 → 토스 계정으로 가입
2. 개발자 콘솔에서 **새 앱 만들기** → 앱 이름 `내일쪽지 뽑기`

## 2단계. 앱인토스 프로젝트 생성 (10분)

내 PC(Windows PowerShell)에서:

```bash
npx create-ait-app tomorrow-note
# 선택: package manager=npm / template=react-ts / TDS=Y / examples=인앱 광고
cd tomorrow-note
```

## 3단계. 이 저장소 코드 이식 (20분)

1. 생성된 프로젝트의 `src/` 를 **이 저장소의 `src/` 로 통째로 교체**
2. `package.json` dependencies 에 `"pretendard": "^1.3.9"` 추가 후 `npm install`
3. 앱 설정 파일(`apps-in-toss.config` 계열)에서:
   - `appName: 'tomorrow-note'`
   - `displayName: '내일쪽지 뽑기'`
   - `primaryColor: '#114e48'`
   - `icon`: 이 폴더의 `app-icon-600.png` 업로드
4. `npm run dev` 로 로컬 확인 → `npm run build` 통과 확인

## 4단계. 광고 SDK 연결 (30분) — 수익의 핵심

`src/lib/ads.ts` 의 mock 4개를 앱인토스 광고 SDK 호출로 교체:

| mock 함수 | 광고 유형 | 노출 지점 |
|---|---|---|
| `showInterstitialBeforeResult` | 전면형 | 쪽지 선택 → 결과 진입 전 |
| `showRewardAdForDetail` | 보상형 | "항목별 운세랑 행운 세트도 볼래요" |
| `showRewardAdForSaveImage` | 보상형 | "결과 카드로 저장하기" |
| `showRewardAdForRetry` | 보상형 | "다른 쪽지도 뽑아볼래요" |

- SDK 문서의 전면형/보상형 함수를 각 mock 자리에 넣고 `Promise<boolean>` 그대로 반환
- **광고 실패 시 true 반환 유지** (사용자 불이익 금지 — 검수 기준)
- 결과 하단 배너: `AdNotice.tsx` 의 `AdBanner` mock 을 배너 컴포넌트로 교체

## 5단계. 푸시 알림 연결 (20분, 선택 — 리텐션 핵심)

`src/lib/subscribe.ts` 의 `subscribeDaily()` mock 을 앱인토스 푸시 권한 요청으로 교체.
매일 아침 8시 발송 문구 제안: **"📮 오늘의 쪽지가 도착했어요. 요정이 기다려요!"**
(1차 출시에서 생략해도 됨 — 앱 내 도착 배너는 이미 동작)

## 6단계. 테스트 & 검수 제출 (30분 + 대기)

1. `npm run build` → 산출물(.ait)을 콘솔 **앱 출시** 메뉴에 업로드
2. 실기기 테스트 **1회 이상** (검수 요청의 필수 조건):
   - 홈 → 운세 선택 → 기분 → 쪽지 → 편지 결과까지 전체 흐름
   - 광고가 각 지점에서 뜨는지 / 닫아도 앱이 정상인지
   - 360px대 작은 폰에서 잘림 없는지
3. **검토 요청** 제출 → 영업일 최대 3일 대기
4. 반려 시 사유 보고 수정 → 재제출 (이 저장소 검수 대비 항목:
   진입 즉시 광고 없음 ✓ / 무료 결과 광고 없이 제공 ✓ / 의료·단정 표현 없음 ✓ / 해요체 ✓)

## 7단계. 출시 후 — 월 30만원 플레이북

**첫 주:** 콘솔에서 실제 수치 확인
- eCPM 이 얼마인지 (계획 가정: 5,000원)
- 사용자당 광고 노출 횟수 (목표 3~4회)
- 재방문율

**수치별 대응:**
| 상황 | 대응 |
|---|---|
| eCPM 낮음 (1~3천원) | 보상형 비중↑ (보상형이 단가 최고) — 상세보기 광고 유지, 배너 위치 조정 |
| 광고 노출 2회↓ | "다른 쪽지도 뽑기" 동선 강화, 상세 진입 카피 A/B |
| 재방문 낮음 | 푸시(5단계) 켜기 — 도착 배너 클릭률은 이미 높게 설계됨 |
| DAU 정체 | 공유 카드 개선 반영 + 토스 콘솔 노출 이벤트 신청 |

**목표 수치 (계획 기준):** DAU 841~1,120명 × 광고 3~4회 × eCPM 5,000원 ≈ 월 정산 30만원

**콘텐츠 주간 운영 (5분/주):** `src/data/resultTemplates.ts` 에 variant 추가만 하면
자동으로 로테이션에 포함. 반응 좋은 운세 카테고리에 집중 추가.
